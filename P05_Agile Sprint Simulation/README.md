# 🚀 Agile Sprint Simulator (P05)

This project demonstrates the implementation of Agile Scrum practices using a CLI-based sprint simulation tool. It covers sprint planning, execution, tracking, and reporting with real-world Agile workflows.

---

## 🎯 Objective

* Understand Agile Scrum framework
* Plan and manage sprints using backlog and velocity
* Simulate daily standups and task tracking
* Analyze sprint performance using burndown charts
* Generate professional sprint reports

---

## 🧠 Key Concepts

* Agile Manifesto & Scrum Framework
* User Stories & Story Points
* Sprint Planning & Velocity
* Daily Standups
* Kanban Board Workflow
* Burndown Charts
* Sprint Retrospective

## 🚀 What This Project Covers

### 🔹 1. Project Initialization

* Load product backlog (20 user stories)
* Configure team (5 members with roles)
* Assign priorities and story points

👉 Example backlog includes banking features like UPI payments and fund transfers 

---

### 🔹 2. Sprint Planning

* Select stories based on **velocity (e.g., 40 points)**
* Prioritize HIGH → MEDIUM → LOW
* Assign tasks to team members

👉 As shown in the lab, sprint backlog may slightly exceed target velocity due to optimization 

---

### 🔹 3. Sprint Execution

* Simulate daily standups
* Track progress of each team member
* Handle blockers during sprint

#### 🧩 Kanban Workflow

* To Do → In Progress → Done

👉 The Kanban board visually tracks task movement across stages (shown on page 9) 

---

### 🔹 4. Burndown Chart Analysis

* Compare:

  * Ideal progress (planned)
  * Actual progress (real execution)

👉 The chart (page 10) shows deviations due to blockers affecting sprint velocity 

---

### 🔹 5. Sprint Retrospective

* Evaluate sprint performance
* Identify:

  * What went well
  * What needs improvement
  * Action items

👉 Example:

* Achieved: ~83% of planned work
* Issues: estimation errors, dependencies 

---

### 🔹 6. Sprint Report Generation

* Generate HTML report including:

  * Sprint metrics
  * Burndown chart
  * Team performance
  * Recommendations

👉 Report is shareable with stakeholders (page 12) 

---
